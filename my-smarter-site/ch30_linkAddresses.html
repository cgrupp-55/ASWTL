Link Addresses           

This page demonstrates the use of link addresses

When a link (e.g., anchor tab with href property) does not reference a specific file, such as a link to the web site of https://www.w3schools.com/, w3school.com Home Page, the browser will assume you want the index.html, file at that site's location.

When linking to another file, use the entire URL, such as https://www.w3schools.com/html/default.asp, such as w3school HTML Home Page"

Note: The above url has a subfolder, HTML, listed in the URL, plus the desired filename is specified as part of the URL's path.

Note: There can be any number of subfolder levels specified in the path.

Note: When the href does not specify a URL's domain name, the browsers will assume the page is referncing a file on the user's computer. But, you still have to specify the directory structure to specify the files's location.

Note: If the location of the page is on a subdirectory that is one level down from the Home page, but on the same level as the location of the web page, the href property must use an instruction in the path to go up one directory, and then come back down to the target directory. The instruction with the path is ../newDirectory/fileName.

Note: If the page is several levels down from the Home page, then you would use as many ../ instructions as necessary to get to the level of the Home page. For example, to go two levels up to get to the Home page, the href property would include these instructions: ../../newDirectory/fileName.
